SECTION 1 — PRODUCT REQUIREMENTS DOCUMENT (PRD)

1.1  Executive Summary

StremFusion v2.0 is a next-generation Windows desktop application that enables seamless Android device mirroring, wireless auto-connection, real-time file drag-and-drop, and full device control — all without requiring any permanent app installation on the phone. It is built on top of open-source technology (scrcpy, ADB) and packaged as a standalone .exe using Antigravity.

The primary upgrade from v1.0 is the elimination of the USB-first requirement: users will be able to pair and connect their Android device entirely over Wi-Fi using Android 11+ Wireless Debugging, with the desktop app handling discovery, pairing, and connection automatically.

1.2  Problem Statement

Existing solutions like Microsoft Phone Link work only on specific OEMs (Samsung, select Android devices), require Microsoft account sign-in, and offer limited file transfer control. scrcpy is powerful but requires manual terminal commands. No existing open-source desktop app provides:

Auto Wi-Fi pairing without USB (for Android 11+)

Native drag-and-drop file transfer from the mirrored screen

A polished, modern desktop UI

A fully portable, no-install .exe for Windows



1.3  Goals & Non-Goals

2.3  Wi-Fi Auto-Connection Flow

The wireless connection flow is the most critical new system in v2.0. It relies on Android 11+ Wireless Debugging (ADB mDNS) and eliminates the need for a USB cable after the initial one-time pairing.

App starts → begins mDNS scan for _adb-tls-pairing._tcp and _adb-tls-connect._tcp services on LAN

Discovered devices shown in UI with IP address and Android model name

User taps 'Pair New Device' → app shows 6-digit pairing code

User enables Wireless Debugging on phone → taps 'Pair with pairing code' → enters code

App calls: adb pair [IP]:[PORT] [CODE] → pairing succeeds

Device fingerprint saved to trusted-devices.json in app data directory

App calls: adb connect [IP]:5555 → device becomes available

scrcpy launched against the Wi-Fi connected device

On next launch → mDNS finds same device → auto-connects in < 10 seconds → no user action needed



2.4  Drag & Drop File Transfer — System Design

Drag-and-drop from PC to phone:

User drags file from PC filesystem onto the scrcpy mirroring window

Electron intercepts dragover and drop events on the window

File path extracted from the drop event

IPC call: file:push with { localPath, devicePath: '/sdcard/StremFusion/' }

Main process executes: adb push [localPath] [devicePath]

Progress streamed via stdout parsing, shown in UI as a progress bar

Completion notification shown in app



Drag-and-drop from phone to PC:

User selects file in the Phone File Explorer panel

User drags it to any PC folder (via Electron's native drag-out API)

IPC call: file:pull with { devicePath, localDestination }

Main process executes: adb pull [devicePath] [localDestination]

Progress tracked and displayed, file lands in destination folder

4.4  Security Architecture

contextIsolation: true — renderer process cannot access Node.js APIs directly

nodeIntegration: false — all Node.js calls go through contextBridge

ADB auth: devices must explicitly authorize the PC (Android security layer)

Trusted device list stored in encrypted electron-store with machine-specific key

No internet connections; all file transfers are local LAN only

Firewall note: port 5555 (ADB TCP) and dynamic mDNS ports must be allowed on local network

6.2  Antigravity Build Process

Antigravity is the AI-powered build tool used to package this app as a standalone Windows .exe. The following steps are performed by Antigravity:

Reads electron-builder.json configuration

Runs Vite production build: npm run build

Compiles Electron main process TypeScript

Bundles /resources/ (scrcpy + adb binaries) into the package

Generates portable Windows x64 .exe (no installation required)

Optionally signs the .exe with a code signing certificate

Outputs: StremFusion-v2.0-Setup.exe (installer) and StremFusion-v2.0-Portable.exe
